# 页面与交互规格

## 1. 导航结构

主导航固定为：

- 报告；
- 当前报告：仪表盘、人工复核、完整核查表、整改任务、输出、审计；
- 系统状态。

没有当前报告时，只显示报告入口和系统状态。普通用户界面不单独设置 profile、ontology、route、evidence kind 或规则管理页面。

## 2. 报告列表 `/reports`

### 主要任务

- 查看已有报告；
- 上传新报告；
- 打开上次报告；
- 查看分析和高风险复核状态。

### 列表字段

- 企业名称；
- 报告年度；
- 文件名；
- 语言；
- 页数；
- 报告状态；
- 最新分析时间；
- 高风险数量与复核进度；
- 最新正式输出版本。

### 状态

- 首次空状态：上传区域作为主内容，不显示虚构指标；
- 加载：稳定表格骨架；
- 上传失败：保留已选文件并显示可读原因；
- 列表为空：显示支持 PDF 和文件保护说明；
- 列表有数据：上传入口收敛为工具栏命令。

用户偏好 `open_last_report` 只保存在浏览器；关闭时始终停留在报告列表。

## 3. 报告信息确认 `/reports/[reportId]/confirm`

系统预填：企业名称、报告年度、语言、PDF 页数、文件名和文件哈希摘要。用户可编辑前三项，页数和哈希只读。页面提供默认关闭的“启用 AI 辅助分析”选项，并明确说明只发送当前 requirement、有限证据片段、证据 ID、PDF 页码和必要 metadata；AI 建议不会覆盖规则或人工结果。

操作：

- 确认并开始分析；
- 返回报告列表；
- 删除未开始分析的上传记录。

企业名称、年度和语言缺失时禁止开始分析。修改自动识别值时写入 `report_metadata_corrected` 审计事件。

## 4. 分析进度 `/reports/[reportId]/progress`

显示八个业务阶段：文件检查、PDF 解析、报告结构识别、GRI 匹配、证据与结论生成、风险分级、AI 辅助分析、结果汇总。

每阶段显示状态、完成数/总数、耗时和错误摘要。页面轮询 run，不展示内部 worker、route 或 ontology 名称。

### 离开条件

- 全部完成：进入仪表盘；
- 部分完成：进入仪表盘并显示失败项高风险提示；
- 全部失败：停留并提供重试和返回报告列表；
- 用户离开页面：后台分析继续。

## 5. 仪表盘 `/reports/[reportId]/dashboard`

### 首屏

- 报告身份：企业、年度、语言；
- 报告状态；
- 高风险复核进度；
- 进入复核主按钮；
- 草稿或正式输出状态。

### 指标

- 系统结论：已披露、部分披露、未知；
- 风险等级：高、中、低；
- 高风险已复核/总数；
- 部分失败 requirement 数；
- 待整改任务数。

### 图表

- 按 GRI 主题的风险分布；
- 按结论的 requirement 分布；
- 复核进度。

图表只用于导航和汇总，不显示“准确率”。

## 6. 三栏复核 `/reports/[reportId]/review`

### 左栏：风险队列

- 默认只显示高风险待复核项；
- 可切换中风险、低风险和全部；
- 支持 GRI 主题、结论、复核状态和关键词；
- 每项显示中文 requirement 名称、风险原因、证据状态和复核状态；
- 排序：分析失败、无证据、证据质量风险、unknown、其他高风险。

### 中栏：核查详情

- GRI 编号与中文业务名称；
- requirement 原文折叠区；
- 规则结论、规则判断依据和规则缺失项，只读且不随人工 snapshot 改写；
- AI 辅助建议、置信度、中文依据、缺失项、证据页和 guardrail 状态，只读；
- 人工当前有效结论；
- 证据页与证据片段；
- 人工判断依据；
- 缺失项；
- 备注；
- 快速通过、保存修改，以及对 AI 建议的采纳、载入后修改、拒绝并保留规则结论。

编辑采用显式保存。切换 requirement 时若有未保存修改，显示保存、放弃或取消切换。

### 右栏：PDF 证据

- PDF 页面渲染；
- PDF 页码与报告页码；
- 上一证据/下一证据；
- 跳转指定页；
- 高亮 bbox 可用时显示，高亮不可用时不伪造位置；
- 选择文本后可替换人工证据片段。

### 快速通过

预设原因：

- 证据与系统结论一致；
- 零披露声明直接有效；
- 数值或比例直接对应；
- omission note 处理正确；
- 其他，选择后要求备注。

### 窄屏

使用 segmented control 切换：风险队列、核查详情、PDF 证据。当前 requirement、编辑草稿和 PDF 页码保存在页面状态中。

## 7. 完整核查表 `/reports/[reportId]/assessments`

493 个独立判断项分页展示，并说明另有 78 个父级上下文项和 6 个方法待确认项；三者合计 577 个标准核查单元。列：GRI 编号、主题、当前结论、复核优先级、复核状态和证据页。

支持筛选、排序和列显示菜单。点击行进入同一 report 的复核工作台并定位 requirement。

默认不显示内部诊断字段。高级列可显示：系统结论（`verdict`）、复核状态（`review_status`）、PDF 页码（`source_pdf_page`）、报告页码（`source_report_page`）。

## 8. 整改任务 `/reports/[reportId]/actions`

任务字段：标题、来源 requirement、优先级、状态、负责人文本、截止日期、整改建议、完成说明。

状态：待处理、进行中、已完成、已取消。第一版不做组织成员管理，负责人是自由文本。

## 9. 输出 `/reports/[reportId]/exports`

显示：

- 生成草稿；
- 正式输出门槛；
- 历史版本；
- 每个版本的复核范围和文件清单。

高风险未完成时禁用正式输出按钮，并显示剩余数量。允许随时生成草稿，草稿必须带标识。

草稿和正式输出必须包含“AI建议未经人工确认时不构成最终披露结论”，同时披露高/中优先级、适用性待判定和实际人工复核范围。

## 10. 审计 `/reports/[reportId]/audit`

按时间倒序显示上传、metadata 修正、分析、部分失败、重跑、复核、批量操作、重开和导出。支持按事件类型和复核人筛选。

审计记录不可编辑和删除。

## 11. 中文业务名称映射

| 内部值 | 普通界面 |
| --- | --- |
| `disclosed` | 已披露 |
| `partially_disclosed` | 部分披露 |
| `unknown` | 待确认 |
| `high` | 高风险 |
| `medium` | 中风险 |
| `low` | 低风险 |
| `pending_review` | 待复核 |
| `reviewed_approved` | 已通过 |
| `reviewed_modified` | 已修改 |
| `evidence_invalidated` | 证据无效 |
| `partially_completed` | 部分完成 |

## 12. 视觉边界

- 参考 `esg-dashboard` 的克制绿色强调色、清晰侧栏、状态 badge、紧凑面板和可扫描表格；
- 页面背景使用中性浅色，风险色只用于状态；
- 卡片圆角不超过 8px；
- 不使用营销 hero、装饰渐变球或大面积单一深色；
- 操作按钮使用 lucide icons；
- 固定工具栏和三栏宽度使用响应式约束，避免内容跳动。
